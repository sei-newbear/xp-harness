# 実装の規約

プロジェクト固有のコード規約・アーキテクチャ方針を、実装するときに効かせるためのスキル。ここに置く方針は薄いサンプルで、プロジェクトは自分の規約に置き換えて使う前提。

「どう進めるか」(テストのリズム・分割) ではなく、「コードが何を満たすか」(規約・構造・命名・コメント) を扱う。

## コメントの方針 (サンプルの立場)

これはこのスキルが持つ一つの立場 (opinion) であり、上書き前提のサンプル。プロジェクトに別の方針があればそちらに従う。

- コメントは基本書かない。コードを綺麗にして、名前で意図を伝える
- 書くのは Why (なぜこうしたか) のみ。What / How はコードそのもので語る

## 関連する規約スキルがあれば参照する

コードの規約とアーキテクチャを別々のスキルに分けていることがある。本文や文脈で関連スキル (アーキテクチャ規約、プロジェクト固有のコード規約スキルなど) の名前が分かるなら、それを参照して規約に従う。
