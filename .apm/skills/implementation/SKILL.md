# 実装の規約とアーキテクチャ

プロジェクト固有のコード規約・アーキテクチャ方針を実装に効かせるためのスキル。「どう進めるか」(リズム・分割) ではなく、「コードが何を満たすか」(規約・構造・命名・コメント) を扱う。

## コメントの方針

- コメントは基本書かない。コードを綺麗にして、名前で意図を伝える
- 書くのは Why (なぜこうしたか) のみ。What / How はコードそのもので語る

## 関連する規約を探して従う

コードの規約やアーキテクチャをまとめたスキルやルールがプロジェクトにあれば、探して読み、従って実装する。
