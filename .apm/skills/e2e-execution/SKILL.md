# E2E の実行手順

E2E テストを実行する手順を扱うスキル。実行環境のセットアップ、実行コマンド、CI 統合、実行が失敗したときの対処 (環境やコマンドの問題) を担う。

E2E spec の書き方 (仕様書性・セレクタ・ヘルパー・テスト独立性) は別スキルの責務。ここは spec を「書く」のではなく「動かす」側だけを扱う。実行が失敗したときも、扱うのは環境やコマンドの問題であって、spec の中身のデバッグではない。

ここに置く手順は薄いサンプルで、プロジェクトは自分の E2E 実行手順に置き換えて使う前提。

## 関連する実行手順スキルがあれば参照する

プロジェクト固有の E2E 実行手順スキルが利用可能なら、その名前を指定して参照し、手順に従う。
