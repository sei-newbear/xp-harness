# 一括配送料見積もり — 要件定義

> このドキュメントは進行中の設計メモ。実装後もそのまま残す（振り返り・PR レビューで参照される）。
> 削除のタイミング・要否は依頼者または外側の仕組みで別途判断する（あなたは削除しない）。

## 1. 背景と動機

運用担当が、複数の宛先へまとめて発送する場面で、宛先ごとに配送料見積もりを 1 件ずつ取得するのは手間。発送計画を立てるとき、複数宛先の費用を一度にまとめて把握したい。

## 2. ユーザーストーリー

- 運用担当が、複数宛先の発送費用をまとめて把握するために、宛先の配列を渡して各宛先の配送料を一度に見積もりたい

## 3. Done（完了基準）

- 宛先の配列を渡すと、各宛先の配送料が配列で返る
- 各配送料は外部の配送料 API から取得した値である（固定値ではない）
- 宛先が 1 件もないリクエストはエラーになる

## 4. スコープ外

- 見積もり結果の保存・履歴表示
- 一括発送の実行（送り状の発行など）
- 配送業者の選択（今回は既存の配送料 API のみ）
