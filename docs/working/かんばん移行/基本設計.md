# ROADMAP のかんばん化 — 基本設計

> このドキュメントは進行中の設計メモ。実装後もそのまま残す。

## 1. ディレクトリ構成

```
kanban/                              # リポジトリ直下。ROADMAP.md の役割を引き継ぐ
├── README.md                       # 読み方の最小案内（カードでない）
├── TODO_tier1_<title>.md           # 着手対象
├── TODO_backlog_<title>.md         # 未 triage の TODO
└── DONE_backlog_<title>.md         # 完了済み

.claude/skills/kanban/              # 改修者向け skill（直コミット、consumer 非配布）
├── SKILL.md
└── board.sh                        # 状況表示スクリプト（skill 専用なので同梱）
```

## 2. カードのファイル名スキーム

形式: `ステータス_優先度_タイトル.md`（全カード 3 部構成で統一）

- **ステータス**: `TODO` / `DONE`
- **優先度**: `tier1` / `backlog`（`tier2`/`tier3` は実際に振り分けたくなったとき足す）。DONE は完了時点の Tier を残す（完了時に tier1 でなければ `backlog`）
- **タイトル**: 機能を表す簡潔な名前。`_` を含めない（区切りはハイフン、日本語可）

パース規約: 先頭 2 つの `_` で 3 分割する（タイトルに `_` がないので安全）。`README.md` 等 `TODO_`/`DONE_` で始まらないファイルはカードでない。

### ID なし・相互参照なし

かんばんはカード独立が基本。安定 ID は持たせず、カード間を ID で参照させない。過去の欠番記録（やめた理由・経緯）も引き継がない。再利用したい判断軸は `philosophy` skill に昇格済み（後述の取り下げ項目参照）。

## 3. カードの中身

frontmatter は持たせない（ステータス・優先度・タイトルは全部ファイル名にあり、二重管理になる）。本文は緩いテンプレ:

```markdown
# <タイトル>

## 背景 / Why
（何の改善か、なぜ要るか）

## 状況
（現状。ROADMAP の「状況」を引き継ぐ）

## 再開時の起点
（何から始めるか、具体 action）
```

きっちり固定はしない。DONE カードは「再開時の起点」が不要なので、完了内容と日付の記録に留める軽い本文でよい。

## 4. 状況表示スクリプト（`board.sh`）

`kanban/*.md` のファイル名を読む → `_` で 3 分割 → ステータス・Tier 別に集計して並べる。**本文は読まない**（盤面はインデックス、中身はファイルを開く）。読み取り専用、状態変更はしない。

ユースケース → コマンドの対応:

| ユースケース | 見たいもの | コマンド |
|---|---|---|
| 次なにやるか（まず着手対象） | tier1 の TODO 一覧 | `board.sh`（デフォルト） |
| tier1 で決まらない / 棚卸し | TODO 全体（tier1 → backlog） | `board.sh todo` |
| 進捗確認 | DONE 一覧 | `board.sh done` |

- デフォルト（引数なし）= tier1 の TODO 一覧。末尾に `backlog N / DONE M`（他がある通知）と `todo`/`done` の案内を 1 行添える
- `board.sh todo` = TODO 全体を tier1 → backlog の順、件数付き
- `board.sh done` = DONE 一覧
- bash（既存 `scripts/setup-dev.sh` と揃う、依存ゼロ）

## 5. `kanban` skill

改修バックログのかんばんを確認・操作する改修者向け skill。発火入口として「次なにやるか」「これ完了にして」「カード追加」「棚卸し」のような場面で効く。本文に持つもの:

- ファイル名スキーム・ステータス・優先度の規約
- 状況確認は `board.sh` を叩く（いつどのコマンドか）
- カード操作は改修者（エージェント）が透過的に行う: 新規追加は Write、ステータス/Tier 変更は `git mv`（ファイル名を変える）
- 責務は「改修バックログ管理」。XP の進め方の規律ではないので consumer に配らない（`.claude/` 直コミット）

- `.claude/skills/release/SKILL.md`: 「ROADMAP TODO 38」3 箇所 → 「kanban の『GitHub Actions で release notes 自動化』カード」相当の表現

- `ROADMAP.md` 本体は削除（役割は `kanban/` が引き継ぐ）
