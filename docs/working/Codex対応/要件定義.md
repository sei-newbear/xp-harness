# Codex 対応 — 要件定義

> このドキュメントは進行中の設計メモ。実装後もそのまま残す（振り返り・PR レビューで参照される）。
> 削除のタイミング・要否は依頼者または外側の仕組みで別途判断する（あなたは削除しない）。

## 1. 背景と動機

現状 xp-harness は Claude Code 専用で、`apm.yml` の `targets` も `claude` のみ。依頼者は本業で Codex を実利用しており、そこでも xp-harness の XP 規律を効かせたい。APM は Codex を配布先 target として公式サポートしているので、xp-harness を Codex でも配れれば配信範囲が広がる。

`cross-agent 対応` をエージェント単位で分割した 2 枚のうちの Codex 側（もう 1 枚は `Cursor 対応`）。「Codex で xp-harness を動かせる状態にする」で自己完結させる。

痛みはまだ Codex 実環境で確かめきっていない（導入初期）。そこで v1 は「実地検証ファースト」= 最小の enablement を配って、実際に Codex で動かして壊れる箇所を観測してから次を直す方針。

## 2. ユーザーストーリー

Codex を使う開発者が、xp-harness の XP 規律（フェーズ駆動・TDD・ペアプロ的レビュー）を Codex 上でも効かせるために、APM で xp-harness を Codex 向けに install できるようにしたい。

## 3. Done（完了基準）

- `apm.yml` の `targets` に `codex` が含まれる
- 配布 skill 12 個すべての SKILL.md frontmatter に `name`（＝ディレクトリ名）がある
- `apm install --target codex` 後、deploy された `.agents/skills/*/SKILL.md` 12 個すべてが `name` を持つ（Codex は `name` 必須・ディレクトリ名フォールバック無しのため、これが無いと skill がロードされない）
- README に Codex の配布フロー（`apm install --target codex` + `apm compile --target codex`）が、compile が Codex では必要な理由付きで記載されている
- `apm install --target codex` + `apm compile --target codex` で skills(`.agents/skills/`) / subagents(`.codex/agents/*.toml`) / hook(`.codex/hooks.json`) / 中央運用ルール(`AGENTS.md`) が生成され、`AGENTS.md` に改修者向け `CLAUDE.md` の内容が混入しない
- `skill-design-style` の `name` ガイドが cross-agent 前提（skill も `name` 明示）に更新されている

## 4. スコープ外

- **Cursor 対応**（別カード `Cursor 対応`）
- **「Claude」ハードコード語 / `AskUserQuestion` / `TaskCreate` の汎用化**: AGENTS.md 冒頭が「Claude Code 運用ルール / あなたは Claude」になる件を含め、汎用化は実地検証で「実際に困る」と分かってから。共有ソースを触ると Claude target 側にも波及するため、target 別出し分けの可否確認を前段に置く
  - 実地検証の優先ポイント（据え置きリスト内の実害の濃淡）: 摩擦が最も濃いのは `git-workflow` の `EnterWorktree`（worktree 自動作成機構、Claude Code 固有ツール名）。Codex には対応物が示されておらず、skill の中核手順が Codex 上で丸ごと空振りになる。据え置きを解くときは、汎用化 or Codex 側の代替手順をここから当てる。摩擦が軽いのは dialogue-principles / basic-design / define-requirements の「(Claude Code なら CLAUDE.md)」等（周辺が汎用フレームで書かれ、例示と分かる程度で通じる）
- **Codex 実環境での動作保証**: skill の自動発火・subagent 委譲・hook 起動が Codex で実際に効くかは、利用者の実地検証に委ねる（導入初期）
- **main.instructions のスリム化 / install 一発化**: 「Codex でも compile 不要にする（中央ルールを skill 側だけで成立させる）」は、中央ルールの再構成を伴う別トラック。既存の main-instructions スリム化カード群と一体で追う

## 5. 制約（Constraints）

- **Claude target を壊さない**（技術）: `name` 追加はディレクトリ名と一致させるので Claude Code では冗長なだけで無害。`.claude/rules/main.md` への従来 deploy も変わらない
- **compile の適用範囲**（運用）: `apm compile --target codex` は Codex 消費者向けの正規手順（出力は `AGENTS.md` と `.codex/` のみで、`CLAUDE.md` を触らない）。**改修者リポジトリでは compile を走らせない** — 実証済の理由: compile の出力ファイルは target で決まり、`--target claude`（および無指定 / `all`）は `CLAUDE.md` を生成・**上書き**する。改修者リポジトリの手書き `CLAUDE.md`（改修者向けルール）を潰すため。「compile 封印」の本体はこれ。codex 向け compile は `CLAUDE.md` を触らないので、消費者の Codex フロー（`--target codex`）は安全
- **消費者の既存 AGENTS.md 上書きリスク**（運用・実証済）: `apm compile --target codex` は `AGENTS.md` を APM が所有・**上書き**する（生成物末尾に「Do not edit manually」）。Codex 利用者は自前の `AGENTS.md` を持つことが多いので、無警告で走らせると失われる。安全経路: 利用者ルールを `.apm/instructions/` 配下に置くと compile が xp-harness の運用ルールと**マージ**して `AGENTS.md` を生成する（実証: 両方の内容が AGENTS.md に入る）。README にこの警告と安全経路を記載する（Done に含む）

## 付録: Cursor も同 PR で最小対応（apm.yml / README 共有のため）

`apm.yml` と README を触るついでに、Cursor も最低限使える状態にした（PR #18 にバンドル）。検証で判明した Cursor の配置:

- skill は `.agents/skills/`（Codex と共通の cross-client location）→ 本 PR の `name` 追加がそのまま効く（追加作業なし）
- subagent は `.cursor/agents/`、hook は `.cursor/hooks.json`
- **中央運用ルールは install が `.cursor/rules/main.mdc` に配る**（Cursor は `.cursor/rules/` を直読み、Claude と同じ構造）→ **compile 不要、install 一発で完結**。Codex のような `AGENTS.md` 上書き問題は起きない

変更は `apm.yml` の targets に `cursor` 追加 + README の Cursor セクションのみ。Cursor 実環境での skill 自動発火・subagent 委譲・hook 起動は Codex 同様に実地検証に委ねる（導入初期）。

## 付録: 検証で確定した Codex 配布の事実（APM 0.25 / Codex 公式ドキュメント）

- **配置**: `apm install --target codex` は skill を `.agents/skills/`（cross-client 共通、Claude だけ `.claude/skills/`）、subagent を `.codex/agents/*.toml`（`name` 注入済）、hook を `.codex/hooks.json`（SessionStart / UserPromptSubmit）に配置する
- **中央ルール**: install は Codex 向けに中央 instruction を配置しない。中央運用ルールは `AGENTS.md` に載る形式で、`apm compile --target codex` が生成する（出力先は `AGENTS.md` と `.codex/` のみ）。Claude は `.claude/rules/main.md` を install が配るので compile 不要、という非対称
- **skill の name 必須**: Codex 公式「SKILL.md must include name and description」。無いとロード失敗・ディレクトリ名フォールバック無し。APM は 0.16 / 0.25 とも codex deploy で name を注入しないため、ソース側で `name` を持たせる必要がある
- **自動発火**: Codex は skill の `description` にマッチすると自動でロード・起動する（Claude Code と同じ description 発火モデル）
- **CLAUDE.md 非混入**: `apm compile` は `.apm/instructions/` の primitive から instruction を作り、リポジトリ直下の `CLAUDE.md` を instruction ソースにしない（実測で改修者向け内容の混入ゼロを確認）
