# slice-tdd: 自走でアウトサイドインとリズムが崩れる

完了 (2026-07-05, v0.11.0)。slice-tdd をゼロベースで書き直し、outside-in を **「各階層で再帰する契約先行の設計」**（外側の契約を先に決めて内側を surface）に据え直した。用語を一般語に統一し、重複を single-source に統合（365→227 行）。

compose+DB のクリーンアーキ sandbox（`harness-verification --compose`、新設）で、文脈ゼロの別セッションに冷えた検証を2回実施:

- 記録の3大崩れ（ボトムアップ / レビュー1回 / E2E 終盤溜め）は **いずれも再現せず**（レビューは各サイクル、E2E は都度、重い docker E2E も溜めず実行）
- 1回目で残った gap「api 内部が内→外のボトムアップ」は、書き直し後の2回目で解消 — 冷えたセッションが usecase のテストを先に書き、port（repository の契約）を先に定義し gateway が後で実装（依存性の逆転）＝バックエンドの中でも契約先行の設計になった

残 refinement は別カード「route↔usecase の受け渡し契約先行」へ。
