# define-requirements の入口をモード合意ゲート化 (深掘り確認の強度調整)

完了 (2026-07-14)。v0.13.0 でリリース。

要件定義フェーズの「★最初に通す」ゲートを、深掘る / そのまま進める の二択から、モード概念で括り直した構造に組み替え:
- 状態は 3 つ (モード未確定 / 対話モード / 自走モード)
- 非破壊な動き (調査・議論) はノーゲート
- ゲートは「帰結ある行動 (設計・実装・変更) の手前」に 1 つだけ、そこでモードを合意
- 調査は未確定のまま進めてよいが、帰結行動の手前でモード合意に戻す (実装への抜け道にしない)

狙い: 重いタスク・調査先行で確認が飛ぶ (合意なしで自走モードに滑り込む) 失敗を防ぐ。あわせて philosophy 判断軸「対話と自走の境界」に対話モード / 自走モードを明示。

harness-verification で 4 シナリオ (調査先行・重い直接依頼・軽い依頼・制約あり) を検証し全 PASS。

※ 元は「深掘り確認の強度調整」という軽い改善として着手したが、モデル化して本体の改修になった。
