# cross-agent 対話/自走モード体験差異 (GPT-5.6 Sol)

## 背景 / Why

xp-harness を GPT-5.6 (Sol) 環境で試したところ、対話モードの体験は Claude より良かった（一度に大量の文章を出さず、一つずつ適切に話してくれた）。一方で自走モードは逆で、Claude は止まらず進めてくれて良いが、GPT はやたら止まってしまい体験が悪かった。同じ skill / instruction を配っていても、モデルの base disposition によって「対話モードで求める段階的開示」と「自走モードで求める止まらない実行」の出方が逆転しうる。

## 状況

- 観測はまだ一回性（2026-07-21、GPT-5.6 Sol での試用）。定量的な再現性・具体的にどの場面で止まったかは未記録
- Claude 側では「対話モードで喋りすぎる」への対処が別途進行中だが、GPT 側は逆に「自走モードで止まりすぎる」問題を抱えている可能性がある。同じモード概念でも、モデルの base disposition によって効かせたい方向が逆になりうる
- 現状 skill / instruction はモデル非依存を前提に書かれている（cross-agent コア対応で `name` 付与や compile 対応はしたが、モデル別の disposition 差異には未対応）

## 再開時の起点

- define-requirements で要件を固める: 「モデル別に自走モードの止まる/進む閾値を変える」方向か、「まず GPT 側でどこがどう止まっているか具体観測を増やす」方向か、対応方針をまず決める
- 具体観測が薄いので、GPT-5.6 (Sol) での自走モード実行 transcript を追加で集めてから harness-verification で事実確認するのも選択肢
