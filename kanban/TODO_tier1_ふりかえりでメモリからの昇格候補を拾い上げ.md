# ふりかえりでメモリからの昇格候補を拾い上げ

## 背景 / Why

xp-harness の方針は「振る舞いはメモリでなくハーネス (skill / CLAUDE.md / 規約) に宿す」。しかしセッション中に自動保存されたメモリには、本来 skill や CLAUDE.md に昇格すべき知見が埋もれたままになりうる。メモリに留まった知見は再現性がなく consumer にも届かないので、ふりかえりのタイミングで棚卸しして昇格させる経路が要る。

## 状況

retrospective skill はセッションのふりかえりから改善案を引き出すが、メモリを走査して「skill / CLAUDE.md 等へ昇格すべきもの」を拾い上げる観点は持っていない。

## 再開時の起点

retrospective skill にメモリ棚卸しの観点を足すか、別の仕組みにするかの要件定義から。メモリの何を見るか (どのエントリが昇格対象か)、昇格先の判断軸 (利用者向け skill / 改修者向け skill / CLAUDE.md)、拾った後の削除までやるか、をスコープとして整理する。
