# setup-dev 再実行漏れの検知支援

## 背景 / Why

`.apm/skills/` / `.apm/agents/` に新規 skill / subagent を追加したとき、`scripts/setup-dev.sh` の再実行 (symlink 作成) が手動手順のため黙って漏れる。実例: PR #15 で追加された handoff-docs skill と handoff-verifier subagent が symlink 未作成のまま約 2 週間放置され、呼ぼうとして初めて発覚した (2026-07-17 に再実行で復旧)。

漏れの検知は機械が得意な突き合わせ (`.apm/` 配下と `.claude/` 配下の差分) なので、支援装置 (ブロックしない気づきの注入) と相性がよい。philosophy の「hook は矯正か支援か」の支援側に該当し、痛みも顕在化済み。

## 状況

- setup-dev.sh 自体は冪等で正しく動く。問題は「追加したら再実行する」の想起が人 / エージェント任せな点のみ
- 今回は手動再実行で復旧済み。仕組みはまだ無い

## 再開時の起点

検知の置き場所を決めるところから (要件定義)。候補: セッション開始時の支援 hook で差分を突き合わせて気づきを出す / git-workflow のセッション開始チェックに 1 手順足す / CI で突き合わせて落とさず警告する。「支援 hook も失敗モードが大きい節目に限る」の規律に照らして、hook 乱立にならない形を選ぶ。
