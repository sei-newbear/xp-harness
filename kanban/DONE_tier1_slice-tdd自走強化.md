# slice-tdd skill の自走強化 (止まる条件精査 + 相談用 subagent の検討)

完了 (2026-06-18)。slice-tdd の「自走と対話の境界」を 5 条件の羅列から判断軸 + 例に組み替えた。

- 2 段ゲート: (1) 実装レイヤー/要件・設計レイヤー (形か意味か = ユーザー観測可能な挙動・データの意味で測る、数では測らない) → 実装レイヤーは自走確定 (2) 要件・設計レイヤーのときだけ手戻りで即停止/完了後報告。デフォルト自走、即停止は例外
- 安全装置: コード revert でも外 (環境・データ・外部) に残る副作用 (データ削除・外部送信・課金・マイグレーション) は無条件即停止。レイヤー・手戻りと独立
- 自走で手段を選ぶとき選好を持ち込まず e2e-execution/implementation を呼んで流儀に従う一文を追加
- 「想定より大きい」は単独の止まる条件として削除 (規模は story-slicing 委譲、価値が立つなら後回し、Done 無効化は仕様ヌケで拾う)
- 「詰まり (3 回失敗)」は技術ブロッカーとして持続可能なペースに据え置き、設計判断の境界と別物と明示

手段 A (止まる条件の精査) で目的を達成。手段 B (相談用 subagent 新設) は採用せず: 既存 reviewer の用途 3 (判断に迷ったときの第三者視点) と重複し、痛みが明確化していない (philosophy 象限 4) ため。重い内部設計判断は code-reviewer に通す形で受け皿を既存装置に寄せた。

止まるべき実例は未収集のため、改修後しばらく事後評価で観察し、「形と判定して自走したが本当は止まるべきだった」事象が出たら 2 段ゲートの線引きを調整する。
