# define-requirements skill に Constraints (制約) を input/output 両面で追加

完了 (2026-07-14)。v0.13.0 でリリース。

define-requirements に制約 (Constraints) を引き出す手順を新設 (技術に限らず、非機能・セキュリティ / コンプライアンス・コスト・期限・組織 / 社内ルール・リソースを対象)。要件定義.md テンプレートに常設「制約」セクションを追加 (該当なしは「特になし」明記)。スコープ外 (交渉可能な除外) と制約 (非交渉のルール) を区別。技術制約は毎回棚卸しせず、要件の実現可能性が論点のときだけ調べて確認する、という引き出し方の使い分けも反映。

harness-verification で「制約あり」シナリオを検証し PASS (要件定義.md の制約セクションに非交渉制約が種類分類つきで記録されることを確認)。
